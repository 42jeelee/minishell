minishell
프로세스와 파일 디스크립터에 대해 배울 수 있을것

준내 다양한 함수를 쓸 수 있다.

쉘을 만들어라!

1. 새 커맨드를 기다리는동안 프롬프트를 출력해라. ok
2. 워킹 히스토리가 있어야한다. ok
3. 올바른 실행파일을 검색하고, 시작합니다.(PATH 기반, 절대, 상대경로 사용) 
4. 전역변수는 1개만 허용.
5. 닫히지 않은 따옴표나 특수문자는 해석하지 마시오. (?)
6. 단일따옴표 안 메타문자를 해석하지 않도록 처리하시오.
7. 큰따옴표 안 메타문자를 해석하지 뭇하도록 처리하시오.($기호 예외)
8. 리다이렉션 구현
	< 은 입력을 리다이렉션 해야한다.
	> 는 출력을 리다이렉션 해야한다.
	<<은 구분기호가 나타날때까지 입력 읽음. 히스토리 업데이트x
	>>추가모드에서 출력을 리다이렉션 해야한다.
9. 파이프문자를 구현해라.
10. $환경변수를 처리해라.
11. $?를 처리해라. $?는 마지막 파이프라인의 종료상태다.
12. ctrl C ctrl D ctrl \ 를 처리해라.
13. interactive mode에서(평소 상태인듯 프로세스 실행중이지 않을때)
	ctrl C를 누르면 새 줄에서 프롬프트가 표시된다.
	ctrl D는 셸을 종료한다.
	ctrl \는 아무것도 하지 않는다.
14. 아래 기본 제공 기능을 구현해야한다.(내장 명령어)
	echo (with -n 옵션)
	cd (상대, 절대경로)
	pwd (옵션 x)
	export (옵션 x)
	unset (옵션 x)
	env (옵션 x)
	exit (옵션 x)

readline함수는 메모리 누수를 야기할 수 있다. 안고쳐도 됨. 그렇다고해서 다른데서 누수터지면 안된다.
서브젝트에 있는것 '만' 해라.
